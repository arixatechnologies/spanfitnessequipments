# SEO Guide

Each indexable route should have a unique title, description, canonical URL, one clear H1, descriptive headings, internal links, and useful original copy.

The application generates sitemap entries for static pages, categories, products, brands, and published blog posts. Robots rules exclude admin/API paths. Structured data identifies the business and supports products, breadcrumbs, blog posts, and FAQs.

After launch:

1. Verify the domain in Google Search Console and Bing Webmaster Tools.
2. Submit `/sitemap.xml`.
3. Inspect the home, contact, category, product, and blog templates.
4. Monitor indexing, duplicate canonicals, broken links, mobile usability, and Core Web Vitals.
5. Maintain consistent name, address, and phone details across the website and local citations.

Use descriptive local filenames and alt text for final images. Compress photography to WebP/AVIF, avoid text embedded in images, and preserve dimensions to reduce layout shift.

Do not create thin city doorway pages. Publish genuinely useful local service content and link naturally among products, categories, brands, blog guides, and contact actions.
